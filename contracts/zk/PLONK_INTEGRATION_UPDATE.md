### 5. How to Deploy

```solidity
// Deploy PLONK Verifier Wrapper
PLONKVerifierWrapper zkVerifier = new PLONKVerifierWrapper();

// Deploy GAT with PLONK verifier
GAT gat = new GAT(address(agentRegistry), address(zkVerifier));

// Or deploy with AgentRegistry (correct order)
GAT gat = new GAT(address(0), address(zkVerifier));
AgentRegistry registry = new AgentRegistry(address(erc6551Registry), address(gat));
```

### 6. Proof Format

The `PLONKVerifierWrapper` accepts the standard `IZKVerifier.Proof` format:

```solidity
IZKVerifier.Proof memory proof = IZKVerifier.Proof({
    a: [uint256(100), uint256(200)],
    b: [[uint256(300), uint256(400)], [uint256(500), uint256(600)]],
    c: [uint256(700), uint256(800)]
});

uint256[] memory publicInputs = new uint256[](1);
publicInputs[0] = 1; // capability hash / agent ID

bool isValid = zkVerifier.verifyProofBool(proof, publicInputs);
```


2. **Option B**: Keep existing verifier, deploy new contracts with PLONK

3. **Option C**: Create proxy/adapter for backward compatibility

### 9. Known Issues

1. **GAT ↔ AgentRegistry Circular Dependency**
   - GAT needs AgentRegistry address in constructor
   - AgentRegistry needs GAT address in constructor
   - Solution: Deploy GAT first with address(0), then AgentRegistry, then update GAT (if upgradable) or use factory pattern

2. **Proof Format Mismatch**
   - Current wrapper uses simplified Groth16-style proof format
   - Full PLONK proof requires additional points (Z, T1, T2, T3, Wxi, Wxiw)
   - For production: Either update interface or create full PLONK proof generator
